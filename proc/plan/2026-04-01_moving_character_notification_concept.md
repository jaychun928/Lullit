# 움직이는 캐릭터 알림(Moving Character Notification) 구현 계획

## 목표
- 고정된 카드 형태의 알림을 벗어나, 캐릭터가 화면 위를 자유롭게 이동하며 스트레칭을 유도하는 생동감 있는 알림 시스템 구축
- 사용자의 시선을 끄는 부드러운 애니메이션과 직관적인 상호작용(클릭하여 닫기) 제공

## 작업 항목

### 1. UI 구조 개편 및 최적화
- [x] **구현 방식 확정 (2026-04-03)**: '알림창(Window) 자체가 움직이는 방식' 채택
    - **결정 사유**: 화면 전체를 자유롭게 횡단하거나 부유하는 연출이 가능하며, 거대한 투명 창을 띄우는 방식보다 시스템 리소스 소모 및 클릭 간섭(Click-through) 이슈에서 유리함.
- [ ] `StretchOverlay` 창 크기를 캐릭터 크기(200x200)로 축소 및 카드 배경 제거
- [ ] `SpeechBubble` 클래스 신설: 캐릭터와 연동되어 나타나는 독립된 말풍선 UI 개발
- [ ] 캐릭터와 말풍선의 Z-order(레이어 순서) 및 투명도 동기화 로직 구현

### 2. 이동 경로 애니메이션(Movement Engine) 개발
- [ ] `QPropertyAnimation`을 활용한 창 위치(`pos`) 변화 로직 구현
- [ ] **캐릭터별 고유 동선(Trajectory) 정의 및 적용**:
    - 캐릭터 데이터(JSON 등)에 이동 패턴(`pattern`)과 가속도(`easing`) 속성 추가
    - 성격에 따른 차별화: (예: 시바견-통통 튀기, 나비-물결 모양 부유, 고양이-슬며시 등장)
- [ ] 시나리오별 경로 프리셋 제작:
    - [ ] **슬며시 등장**: 구석에서 나타나 중앙으로 이동 후 정지
    - [ ] **화면 횡단**: 좌측 끝에서 우측 끝으로 일정한 속도로 횡단
    - [ ] **둥둥 부유**: 특정 구역 내에서 사인파(Sine wave) 형태로 부유
- [ ] 애니메이션 재생 시간과 알림 유지 시간(15초)의 정밀한 동기화

### 3. 상호작용 및 UX 강화
- [ ] **클릭 포획 로직**: 이동 중인 캐릭터를 마우스로 클릭 시 즉시 닫기 기능 강화
- [ ] **근접 반응(선택 사항)**: 마우스 커서가 캐릭터 일정 거리 이내로 접근 시 회피 또는 모션 변화
- [ ] **등장/퇴장 연출**: 페이드 인/아웃과 위치 이동을 결합한 자연스러운 등장 및 퇴장

### 4. 기술적 검증 및 최적화
- [ ] 멀티 모니터 환경에서의 좌표 계산 및 이동 범위 제한 로직 검증
- [ ] 이동 중 CPU 점유율 최적화 (GIF 프레임률 및 애니메이션 업데이트 주기 조절)
- [ ] Win32 API 클릭 통과(Click-through) 속성이 이동 중에도 안정적으로 유지되는지 확인

## 향후 기대 효과
- **브랜딩 강화**: '나를 달래주는 스트레칭 요정' 컨셉을 시각적으로 완성
- **사용자 참여도 상승**: 정적인 알림보다 높은 시각적 자극으로 스트레칭 수행 유도
- **비간섭성 유지**: 창 크기 최소화를 통해 실제 작업 영역 침범을 더욱 줄임
