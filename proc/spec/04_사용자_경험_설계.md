# 04. 사용자 경험 설계 (UX Flow / Screen Flow / Use Case)

### Window Specification
- **메인 윈도우**: `980x860` 고정 크기 (설정 항목의 텍스트 잘림 방지 및 시인성 확보를 위해 확대).
- **알림 오버레이**: `620x340` 고정 크기 (200x200 캐릭터 이미지/GIF 포함).

### Navigation Flow
- 앱 실행 시 항상 우측 하단 **시스템 트레이 아이콘**에 상주하며, **메인 홈 윈도우**가 등장한다. (메인 윈도우 우측 상단 X 버튼 클릭 시 백그라운드 숨김 처리)
- 사이드바 메뉴 클릭 시 해당 페이지는 항상 **초기 상태**로 리셋된다.
  - 예: 스트레칭 상세 화면에서 메뉴를 나갔다 돌아오면 다시 부위 선택 목록이 보인다.
- 설정 변경 후 **[저장] 버튼**을 누르지 않고 다른 메뉴로 이동할 경우, 변경사항은 **즉시 폐기**되며 다시 진입 시 기존 저장된 값으로 UI가 동기화된다.
- 고해상도(QHD)부터 저해상도(FHD)까지 모든 환경에서 UI가 일정한 물리적 비율을 유지하도록 **동적 스케일링** 경험을 제공한다.
- **테마 설정**: 사용자의 취향에 따라 '밝은 테마' 또는 '어두운 테마'를 수동으로 선택하여 적용 가능.

### Alert Flow
- 사용자가 코딩, 문서 작업 등을 집중해서 수행하여 **Fatigue Score**가 차오른다.
- 임계점(설정된 주기) 달성 순간 휴식 타이밍으로 판단.
- 오버레이가 스르륵(Fade-in) 나타나며 시스템 최상단에 뜨되, 뒤편 작업 화면 조작을 방해하지 않는다.
- **알림 멘트**: 스트레칭 방법 대신 "휴식 권장 멘트"를 우선 노출하여 사용자의 정서적 환기를 돕는다. (예: "잠시 숨을 고를 시간이에요.")
- **캐릭터 이미지**: `character` 폴더의 GIF 애니메이션 또는 이미지를 `200x200` 크기로 표시.
- **유지 시간**: 15초 동안 유지 후 자동 소멸. 우측 상단 **닫기 버튼(X)**을 통해 즉시 닫기 가능.

### Detailed Screens UX
1.  **홈 화면 (Home Dashboard)**:
    *   **타이포그래피**: **상단 메인 제목(32px, 800)**, 부제목(22px, Bold)으로 위계 강화.
    *   **핵심 지표**: **프로그레스 바**(높이 16px, 그림자 제거, 8px 둥근 모서리)를 통한 시각화.
    *   **이용 팁**: "💡 Lullit 이용 팁 확인하기" 버튼(Hover 시 초록 배경)을 통한 팝업 노출.
2.  **사이드바 네비게이션**:
    *   **레이아웃**: 모든 메뉴와 하단 버튼(의견 보내기, 종료)이 **상단 고정(AlignTop)** 되어 일정한 간격을 유지. 페이지 전환 시 레이아웃 흔들림 방지 로직 적용.
    *   **메뉴 구성**: 홈, 스트레칭 클래스, 추천 샵, 알림 상세 설정, 고급 설정, 마이페이지 순차 배치.
    *   **하단 메뉴**: "앱 개선 의견 보내기", "Lullit 완전히 종료하기"를 일반 메뉴와 동일한 간격으로 하단에 배치.
    *   **창 제어**: 우측 상단 X 버튼(WindowCloseBtn, (너비-70, 27) 위치)으로 메인 윈도우 숨김.
