# 08. 기술 환경 (Tech Stack / Architecture / Environment)

### 1. 개발 스택 (Tech Stack)
- **Language**: Python 3.13.x
- **UI Framework**: PySide6 (Qt for Python) 6.10.2
- **Icon Format**: SVG (Scalable Vector Graphics)
- **Data Storage**: JSON (app_config.json)

### 2. 아키텍처 설계 (Architecture)
- **MVC 패턴**: 핵심 로직(FatigueEngine), 데이터(ConfigManager), UI(MainWindow) 분리.
- **Single Instance**: QLocalServer를 활용한 중복 실행 방지 및 기존 창 활성화 로직.
- **Lazy Initialization**: 
  - 메인 윈도우의 각 페이지는 사용자가 처음 요청할 때 동적으로 생성하여 초기 메모리 점유율 최소화.
  - 리소스(폰트, 아이콘)는 필요한 시점에 로드하거나 메모리 내 캐싱 처리.

### 3. 기술적 특이사항 (Advanced Tech)
- **동적 스케일링 (Dynamic UI Scaling)**:
  - `QHD (2560x1440)` 해상도를 1.0(Base)으로 설정하여 고해상도 환경의 물리적 UI 크기 유지.
  - `ui_utils.py`의 `dp()` 함수를 통해 픽셀 기반 수치를 실시간으로 보정하여 저해상도(FHD 등) 대응.
  - QSS 스타일시트 내 `px` 단위를 정규표현식(`re.sub`)으로 자동 치환하여 전역적인 스케일링 적용.
- **DPI 인식 (DPI Awareness)**: 
  - `AA_EnableHighDpiScaling` 속성을 통해 OS 배율 설정을 앱이 자동으로 인식하여 텍스트 흐림 방지.

### 4. 성능 최적화 (Optimization)
- **Memory Management**: 
  - **상태 초기화**: 메뉴 전환 시(`switch_page`) 상세 뷰 위젯을 `deleteLater()`로 즉시 파괴하여 메모리 누수 방지.
  - **적극적 GC**: 페이지 전환 및 테마 변경 시 `gc.collect()`를 명시적으로 호출하여 미사용 리소스 반환.
  - **이미지 최적화**: 트레이 아이콘 등은 메모리 점유율을 고려하여 로드 직후 리샘플링 및 원본 객체 삭제.

### 4. 실행 및 배포 (Run & Build)
- **실행 런처**: `py -3.13 src/main.py` 권장.
- **인코딩 정책**: 모든 소스코드 및 리소스(QSS, JSON)는 **UTF-8 (BOM 없음)** 형식을 엄수.
- **환경 변수**: `src` 디렉토리를 `PYTHONPATH` 최상단에 배치하여 모듈 참조 보장.
- **빌드 방식**: PyInstaller `onedir` (표준 디렉토리) 방식 채택.
  - 단일 파일(`onefile`) 방식보다 백신 오탐지율이 낮고 실행 속도가 빠름.
- **설치 프로그램**: Inno Setup 활용.
  - 관리자 권한(`PrivilegesRequired=admin`) 필수 요구.
  - 빌드된 디렉토리 내 모든 파일을 하위 폴더 구조를 유지하며 설치 경로에 배포.

### 5. 안정성 및 호환성 강화 (Stability & Compatibility)
- **필수 런타임**: Visual C++ 2015-2022 재배포 가능 패키지 (Redistributable) 의존.
  - 설치 프로그램 시작 시 사용자에게 안내 메시지 표시.
- **백신 예외 대응**: 
  - `pynput` 라이브러리를 사용한 키보드/마우스 입력 감지 로직은 보안 프로그램에 의해 차단될 가능성이 높음.
  - `onedir` 빌드와 코드 서명(추후 도입 권장)을 통해 오탐지 최소화.
- **중복 실행 방지**: `AppMutex` (Inno Setup) 및 `QLocalServer` (Python)를 활용하여 설치 및 실행 단계에서의 충돌 방지.

### 6. 보안 및 리소스 보호 (Security & Resource Protection)
상업적 활용 및 수익 구조(광고 등) 보호를 위해 리소스의 무단 변경을 방지하는 전략을 단계적으로 도입한다.

- **1단계: Qt 리소스 시스템 (RCC) 도입**
  - 이미지, GIF 등 핵심 자산을 독립된 파일로 두지 않고, `pyside6-rcc`를 통해 파이썬 코드(.py) 내부의 이진 데이터로 컴파일하여 포함한다.
  - 사용자가 설치 폴더 내에서 리소스를 직접 식별하거나 교체하는 것을 원천 차단한다.
- **2단계: 리소스 무결성 검사 (Integrity Check)**
  - 실행 시 핵심 리소스의 해시값(SHA-256 등)을 대조하여, 변조가 감지될 경우 앱 실행을 중단하거나 기본 리소스로 강제 복구하는 로직을 구현한다.
- **3단계: 서버 기반 동적 리소스 로딩 (Server-side Loading)**
  - 상업용 서비스 단계에서는 로컬 파일 대신 서버에서 암호화된 스트림으로 이미지를 받아와 메모리상에서만 로드한다.
  - 이를 통해 광고 콘텐츠의 실시간 교체 및 노출 횟수 집계를 보장하며 사용자 측의 조작을 방지한다.
